# Prompt Studio UX改善仕様 v0.2

## 1. 目的

Prompt Studio v0.1は、内部設計としては適切であるが、ユーザーが日常的に使用する開発ツールとしては概念が多く、操作回数が多いという課題がある。

v0.2では、

> 「プロンプトを管理するツール」

ではなく、

> 「人格やプロンプトを試行錯誤するためのIDE」

としてUXを再設計する。

---

# 2. 基本思想

ユーザーが行いたいことは以下のループである。

```text
編集
 ↓
生成
 ↓
実行
 ↓
確認
 ↓
編集
```

このサイクルを10秒以内で回せることを最優先とする。

# 15. UX設計原則

Prompt Studioは、

> Promptを管理するツール

ではなく、

> Promptを試して育てるためのIDE

として設計する。

そのため、

* Buildを意識させない
* Exportを意識させない
* Layerを意識させない
* ファイル管理を意識させない

一方で、

* 編集
* 実行
* 比較
* 修正

という試行錯誤サイクルを最優先に設計する。

最終目標は、

```text
編集
 ↓
自動生成
 ↓
Run
 ↓
結果確認
 ↓
修正
```

を10秒以内で反復できる体験を実現することである。

# 16. 実装判断（v0.2）

以下の方針で実装する。

## 16.1 focus モードでの Component 一覧

**左スライドオーバー**で開閉する。

* デフォルト（focus）では Component ペインは非表示
* ツールバーのボタンでスライドイン
* ファイル選択後は自動で閉じる（任意・実装時に調整可）

## 16.2 Export Prompt（workspace 出力）

主フローからは外し、**メニュー + キーボードショートカット**に退避する。

* メニュー（⋯）内に「Export to workspace」
* ショートカット: `Ctrl+Shift+E`

## 16.3 ファイルの論理名

`identity.md` →「基本人格」等の表示名は **v0.3 候補**とし、v0.2 ではファイル名をそのまま表示する。
