# Prompt Studio 仕様書 v0.1

## 1. 概要

Prompt Studio は、LLM向けプロンプトをレイヤー構造で管理・編集・組み立て・テストするためのブラウザベースの開発環境である。

本ソフトウェアは配信ランタイムとは完全に独立して動作し、プロンプト開発に特化する。

目的は以下とする。

* プロンプトを部品化する
* 用途ごとに組み合わせを変更できる
* Markdownで管理しやすくする
* Gitとの相性を良くする
* オープンソースで拡張しやすくする

---

# 2. 基本方針

Prompt Studioは以下の責務のみを持つ。

* プロンプト編集
* レイヤー管理
* プロンプト組み立て
* 完成プロンプトのプレビュー
* LLMによる動作確認（任意）

以下は責務外とする。

* 配信ランタイム
* KVキャッシュ生成
* 推論高速化
* 配信機能

# 4. layers

layersディレクトリにはレイヤーごとのMarkdownを配置する。

例

```text
persona/
    identity.md
    values.md
    speech.md
```

Markdownにはプロンプト本文のみを記述する。

見出しやコメントは自由に記述してよい。

---

# 5. layers.yaml

layers.yamlはレイヤー一覧を定義する。

例

```yaml
layers:
  - id: system
    name: System
    description: System Prompt

  - id: persona
    name: Persona
    description: Character

  - id: memory
    name: Memory

  - id: style
    name: Style
```

役割

* GUIへの表示
* レイヤー一覧管理
* レイヤー説明

プロンプトの並び順は管理しない。

---

# 6. Export

exportsディレクトリには用途ごとの組み立て設定を配置する。

例

```yaml
name: AITuber

build:
  - layer: system
    prompts:
      - role.md
      - safety.md

  - layer: persona
    prompts:
      - identity.md
      - speech.md

  - layer: style
    prompts:
      - concise.md
```

Exportは

* 使用するレイヤー
* 使用するMarkdown
* 結合順

を定義する。

同じMarkdownを複数Exportで再利用できる。

---

# 7. Prompt Builder

Prompt BuilderはExportを読み込み、

Markdownを順番に結合して最終プロンプトを生成する。

処理フロー

```
Export

↓

Layer読込

↓

Markdown読込

↓

結合

↓

完成Prompt
```

完成Promptは右側プレビューへ表示する。

# 9. LLM実行

LLMは任意機能とする。

LLM未設定でもPrompt Studioは利用可能。

設定された場合のみ

```
Prompt

↓

LLM

↓

Response
```

を実行する。

# 15. 設計思想

Prompt Studioは「プロンプトを書くエディタ」ではない。

レイヤー単位でプロンプトを管理し、用途に応じて組み立て、LLMで確認するための「レイヤー型プロンプトIDE」である。

Markdownを中心としたシンプルな構成とし、Git管理・オープンソース開発・再利用性を重視する。

配信ランタイムとは責務を明確に分離し、Prompt Studioはプロンプト開発環境、配信ランタイムは実行環境として運用する。
